# Contributing to EcoSmart AI

## Branch Naming
- `feature/waste-scan-ui`
- `feature/ai-classification-service`
- `fix/recycler-api-timeout`
- `chore/update-dependencies`

## Commit Message Format (Conventional Commits)
```
feat: add image upload to waste scan
fix: handle null response from AI service
chore: update eslint config
docs: update API endpoint docs
test: add unit tests for wasteService
```

## Pull Request Process
1. Branch off `develop` (never directly off `main`)
2. Keep PRs focused — one feature or fix per PR
3. Fill in the PR template completely
4. Request review from at least one team member
5. All CI checks must pass before merging

## Code Review Etiquette
- Be kind and constructive
- Approve only when genuinely satisfied
- Use "Request changes" for blockers, comments for suggestions
